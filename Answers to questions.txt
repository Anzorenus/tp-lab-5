..В чем разница между передачей в функцию параметра const string& и string& ?..
передача const string& означает, что строка по ссылке не будет изменяться внутри функции, 
когда string& этого не запрещает

..Как работает механизм начисления баллов Студентам?..
У каждого студента есть свой вектор оценок и метод add_mark(), который добавляет оценку в вектор. 
Поэтому, либо деканат, либо группа, либо сам студент может инициировать добавление оценок.
В то же время в деканате есть метод addGrades, который проходит по всем студентам в принадлежащих группах
и начисляет рандомные оценки.


..Как происходит процесс перевода Студента из Группы в Группу?..
В функцию передаются название группы из которой нужно перевести студента, его id и 
название группы,  в которую студента нужно перевести. Внутри функции проверяется, есть ли действительно такие группы, 
и если есть, то инициируется метод kick_student() из своей первоначальной группы. kick_student возвращает указатель на студента, если такой 
имеется в группе и удаляет его из группы, либо nullptr. Далее, если студент нашелся, то ему предписывается новая группа и он зачисляется в нее.

..Как можно определить отношения между классами Деканат и Группа? Почему?..
Класс деканат является дружественным классу группа, но не наоборот. Это нужно для того, чтобы предоставить деканату доступ к приватным методам группы и возможность создавать группы, 
в то время как эти методы не должны быть доступны из других классов (группы не могут существовать вне деканата). Если говорить про отношения на языке ООП, то
это композиция, так как класс деканат является коллекцией классов группа и при уничтожении деканата, уничтожатся и все принадлежащие ему группы. 

..В чем состоит принцип построения сценария работы Деканата для main?..
Функция main может использоваться для многих сценариев, например:
	1. Чтобы запустить приложение, т.к main() является отправной точкой, откуда программа начинает свое исполнение. Иначе было бы не ясно, откуда начинать программу.
	   1.1 Чтобы протестировать работу реализованных функций. 
		1.1.1 Чтобы протестировать работу тестов до пуша в репозиторий и долгого запуска тревиса :))
	   1.2 Чтобы запустить веб сервер, либо десктопное приложение.

В моем случае, функция main использовалась для тестирования приложения в общем. Из автоматически сгенерированного файла с людьми генерировались файлы групп студентов, 
из этих файлов создавался новый деканат и тестировались все его функции.
